# Contributing

## Bugs

Report bugs in issues. Include a clear description, steps to reproduce, expected behavior, actual behavior, and any relevant logs or screenshots.

## Features

Discuss feature ideas in issues before implementing them. Use the issue to align on scope, tradeoffs, and acceptance criteria before opening a pull request.

## Pull requests

Contribute fixes and features through pull requests. Create a feature branch from `main`, keep it focused on one topic, and open the pull request back to `main`.

## Review etiquette

Use [Conventional Comments](https://conventionalcomments.org/) when giving review feedback so intent is explicit and discussion is easier to act on.

## Keep changes focused

Keep contributions small enough that review and discussion stay focused. Split unrelated work into separate issues and pull requests.
